## PART 4: FINAL AGENT ROSTER — REDESIGNED

### Design principles (derived from all transcripts + infrastructure audit)

1. **Fewer agents, more skills.** Agents are expensive (per-invocation token cost, complex routing). Skills are cheap (one-shot prompt templates, reusable). Convert most "agent" functions to skills.
2. **Skills over agents for repeatable work.** If the task is the same pattern every time (editorial pitch, DM blitz prep, thumbnail generation), it's a skill. If it requires judgment + cross-referencing + disagreement surfacing, it's an agent.
3. **Build for Cowork, not Claude Code.** Ethan lives in Cowork. Skills and slash commands need to work there.
4. **ADHD-first design.** One entry point. No routing tables. No "remember to call X."

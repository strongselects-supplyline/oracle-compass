# PER-RELEASE PIPELINE — A to Z
## past.El noir Records · Canonical Process (May 9, 2026)

This is the single-source-of-truth for every step from idea to post-release compound. Every department touches every release. Nothing ships without clearing every phase below.

Estimated total cycle per release: **14–21 days** (from final mix to post-release compound).
Estimated total cycle per track (idea to master): **variable** (batch by stage, not by track).

---

## PHASE 0: CREATION (variable timeline)

**Tools:** FL Studio, Suno (demo ideation only), Auto-Tune Pro (graph mode)

| Step | Action | Done When |
|------|--------|-----------|
| 0.1 | **Idea capture** — voice memo, melody, chord progression, vibe reference | Idea exists in any form |
| 0.2 | **Production topline** — beat/production draft in FL Studio | Instrumental draft plays start to finish |
| 0.3 | **Re-production** — refine arrangement, sound selection, structure | Beat is "would-release" quality |
| 0.4 | **Recording** — vocal takes (batch all recording across tracks) | Raw vocals captured, all takes done |
| 0.5 | **Rough vocal mix** — comp → manual de-breath/click removal → second comp → AT Pro graph-mode tuning → bake final comp | Clean vocal comp baked |
| 0.6 | **Rough mixing** — levels, panning, FX chain, space | Rough mix plays well on phone speaker + car |
| 0.7 | **Cyanite analysis** — upload rough mix, check genre/mood confidence scores | Scores logged, genre tags confirmed |
| 0.8 | **Second draft** — adjust based on Cyanite + ears (does the data match intent?) | Adjustments applied or consciously rejected |
| 0.9 | **Final mix / polish** — last 10% (automation, ear candy, transitions) | Mix is final |
| 0.10 | **Master** — Illangelo chain, -9 to -11 LUFS integrated streaming master + -14 LUFS sync master | Two masters exported (streaming + sync) |

**Batch rule:** Work in stages, not per-track. Record everything, THEN mix everything, THEN master everything. Interleaving kills momentum.

**Suno workflow:** When stuck on melody/arrangement, feed the idea to Suno as a demo prompt. Use the output as a sparring partner — something to beat, not something to use. Writer's block = solved.

## PHASE 1: PRE-UPLOAD (2–3 days)

**Tools:** Photopea, CapCut, Amuse, Oracle Compass

| Step | Action | Done When |
|------|--------|-----------|
| 1.1 | **Cover art** — design or photograph, finalize in Photopea (3000x3000 min for Amuse) | Final PNG exported |
| 1.2 | **Spotify Canvas** — isolate layers in Photopea → keyframe in CapCut (1080x1920, 6-8 sec loop) → export MP4 | Canvas MP4 under 20MB, loops clean |
| 1.3 | **Metadata prep** — song title, featured artists, genre tags (R&B / Alt-R&B, NOT Pop), ISRC (new or carry), release date | Written down, ready to paste into Amuse |
| 1.4 | **Upload to Amuse** — all files, metadata, cover art. Set release date (7 days minimum for ingestion) | Amuse confirms "Processing" |
| 1.5 | **Pull ISRC** — once Amuse assigns it, save to releases.ts + catalog_intelligence_matrix.json | ISRC recorded in both locations |
| 1.6 | **Update Oracle Compass** — update releases.ts status to "upload_pending", verify phaseMap.ts dates | Oracle reflects correct state |

**Genre tagging rule:** NEVER blanket-tag as Pop. Use R&B / Alternative R&B on Amuse. This directly affects which algorithmic pool Spotify places the track in.

**Canvas pipeline (confirmed May 9, 2026):**
1. Open cover art in photopea.com
2. Isolate elements to separate layers (subject, background, text, moving elements)
3. Content-Aware Fill holes in background layer
4. Export each layer as transparent PNG
5. Import all PNGs into CapCut desktop (free version, 1080x1920 project)
6. Stack layers, keyframe scale + position + rotation per layer (parallax = layers moving at different speeds)
7. Add atmosphere overlay (dust/fog from Pexels, Screen blend mode, 15-20% opacity)
8. Export MP4, upload to Spotify for Artists → Music → [Track] → Canvas

**Time estimate:** ~2 hours per canvas (first one took 3h learning; now repeatable at 2h).

## PHASE 2: PRE-RELEASE SPRINT (T-7 to T-0)

**Tools:** Spotify for Artists, Instagram, CapCut, Oracle Compass Kill List

This is the 8-day content sprint. Every day has a specific deliverable.

| Day | Action | Department |
|-----|--------|------------|
| T-7 | **Batch prep** — shoot/capture 5-7 content pieces. BTS studio footage, direct-to-camera, aesthetic world-build clips. | Content |
| T-5 | **Teaser** — 15-sec clip (hook or production moment). Post IG Reel + TikTok + Shorts. No song title yet — mystery. | Content |
| T-5 | **Spotify editorial pitch** — submit via Spotify for Artists (28-day window preferred, 7-day minimum). Genre, mood, description, instruments. | Marketing |
| T-3 | **World build** — aesthetic post that sets the vibe without directly promoting. 2 AM drive, studio late night, Milwaukee. | Content |
| T-3 | **Pre-save link** — generate via Amuse or linkfire. Add to IG bio + link in bio tool. | Marketing |
| T-1 | **Announce** — cover art reveal + release date + pre-save CTA. Carousel or single image. Caption: date, one sentence, pre-save link. | Content |
| T-1 | **DM blitz prep** — draft 3 tiers of messages: Core 50 (personal), Warm 100 (story viewers), Cold 50 (mutual followers). | Social |
| T-0 | **Release day** — post release announcement (Reel or carousel). Execute DM blitz. Upload Canvas to Spotify for Artists. Update IG bio link. | All |
| T-0 | **Story blitz** — 3-5 stories throughout the day (listening reaction, lyric highlight, thank you). | Content |

**Content creation order:** Create NATIVE on IG → export clean → upload TikTok → upload Shorts. Never reverse.

**Paid trigger criteria (RIGID, do NOT spend without proof):**
- Reel sends/reach ≥ 3% → eligible for Meta boost ($50 test, 3 days)
- Release week → Marquee eligible ($100 minimum, "Previously Engaged" audience, EP + first vault single only)
- 7-day save rate > 3% → eligible for Discovery Mode (free, 30% royalty trade)
- Kill at $0.10+ cost-per-stream after 48 hrs

**Budget:** $250/month HARD cap. $50 test protocol per campaign. NEVER spend without organic proof.

## PHASE 3: POST-RELEASE COMPOUND (T+1 to T+7)

**Tools:** Spotify for Artists, Oracle Compass, Instagram

| Day | Action | Department |
|-----|--------|------------|
| T+1 | **Momentum** — post second content piece (different angle than release post). Check 24-hr stream count. | Content + Marketing |
| T+1 | **Community trace** — reply to all comments, DM everyone who shared/saved, thank early listeners. 15 min, SET TIMER. | Social |
| T+3 | **Compound** — third content piece. If sends/reach ≥ 3%, trigger Meta boost. Check 72-hr save rate. | Content + Marketing |
| T+3 | **Data pull** — Spotify for Artists: streams, saves, save rate, playlist adds, skip rate, audience demographics. Log to brain. | Marketing |
| T+5 | **Compliance Thursday** — ASCAP registration, MLC registration, Songtrust registration. Always the Thursday AFTER release, never before. | Revenue |
| T+7 | **Week 1 data review** — full data pull. Save rate target: 3%+. If below 2%, the track needs content fuel not paid spend. | Marketing |
| T+7 | **Catalog refresh** — run `brain/refresh-catalog.mjs` to update popularity scores in catalog_intelligence_matrix.json | Revenue |

**Compliance registrations (every track, every time):**
1. ASCAP — song registration (writer share)
2. MLC — mechanical license (streaming royalties)
3. Songtrust — global publishing admin
4. SoundExchange — if not already registered for artist

**Post-release content ideas:**
- Lyric breakdown / meaning behind the track (Talk to 'Em format)
- Studio BTS of a specific production decision
- Fan reaction compilation (once saves/shares accumulate)
- Acoustic/stripped version snippet
- Meme account seeding (unconnected account, trending memes with track as background audio)

## PHASE 4: SYNC PREP (parallel, post-master)

**Tools:** FL Studio (stems export), Photopea (one-sheet), Songtradr/Musicbed/Artlist

| Step | Action | Done When |
|------|--------|-----------|
| 4.1 | **Export stems** — vocals, drums, bass, melodic, FX (separate, full-length, -14 LUFS) | Stems folder exported per track |
| 4.2 | **-14 LUFS master** — separate from streaming master (-9 to -11 LUFS integrated). Sync supervisors need broadcast-standard loudness. | -14 LUFS WAV exported |
| 4.3 | **Instrumental** — full instrumental render, mastered | Instrumental WAV exported |
| 4.4 | **One-sheet** — track title, BPM, key, mood tags, genre, duration, contact, PRO info | PDF or PNG, one page |
| 4.5 | **Platform submission** — upload to Songtradr, Musicbed, Artlist with metadata + stems | Submitted and processing |

**Sync reality check:** A single placement ($500-$5,000) = 100K-1M stream equivalent in revenue. This is the highest-leverage revenue activity after the music is made.

## PHASE 5: ORACLE COMPASS UPDATES (throughout)

**Files to update per release:**

| File | What Changes |
|------|-------------|
| `lib/releases.ts` | Status transitions: unreleased → upload_pending → live. ISRC. Dates. |
| `lib/phaseMap.ts` | Add/adjust PHASE_MAP entries for the release window |
| `lib/killList.ts` | Release-specific tasks auto-derive from releases.ts — verify they fire correctly |
| `lib/studioData.ts` | Track metadata (BPM, key, Cyanite scores) if not already present |
| `brain/catalog_intelligence_matrix.json` | Post-release: ISRC, popularity score, streaming data, Cyanite final analysis |

**Version bump:** Increment `RELEASE_DATA_VERSION` in releases.ts after any default change to force IndexedDB re-seed on all devices.
